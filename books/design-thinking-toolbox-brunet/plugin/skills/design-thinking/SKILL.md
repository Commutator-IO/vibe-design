# Design Thinking — orchestrator

Guide the user through a user-centered innovation project using the book's structure: a **double-diamond** process that alternates divergence (explore widely) and convergence (decide), across six sets of tools. Route to the matching phase skill and keep the work anchored to a real user and a real problem.

## The double diamond → the six phases

| Diamond | Phase | Skill | You produce |
|---------|-------|-------|-------------|
| Setup | Fundamentals | *(this skill)* | Framing, team, space, mindset |
| Discover (diverge) | Immersion & empathy | `immersion-empathy` | Field understanding of users |
| Define (converge) | Analysis | `analysis` | Personas, journeys, a reframed challenge |
| Develop (diverge) | Ideation | `ideation` | A shortlist of ideas |
| Develop→Deliver (converge) | Prototyping & experiments | `prototyping` | Tested low-fi prototypes |
| Deliver | Implementation | `implementation` | A viable plan to ship and scale |

## When to use design thinking
It fits problems needing **innovation** where the answer isn't known. A strong solution sits at the intersection of **desirable** (users genuinely want it), **viable** (economically sustainable), and **feasible** (technically buildable) — plus a responsible, sustainable lens. If the problem is well-defined with a known solution, a straight plan beats design thinking.

## Set up before diverging
- **Mindset** — user-centered, bias to action, embrace ambiguity, defer judgment, "fail fast to learn fast," build on others' ideas.
- **Multidisciplinary team** — mix skills, roles, and perspectives; diversity is the engine.
- **A modular space** — walls for stickies, room to move; works in person or remote.
- **A facilitator** — someone to hold time, energy, and the process.
- **POC → MVP → MAP** — plan to prove the concept, then a minimum viable product, then a minimum awesome product.

## How to run it
Start with `immersion-empathy` unless the user already has field research. Move one diamond at a time, alternating diverge/converge, and loop back whenever a later phase reveals you framed the wrong problem — the process is iterative, not linear. Restate the user and the challenge at each hand-off.

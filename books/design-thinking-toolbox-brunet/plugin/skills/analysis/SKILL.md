# Analysis (Define)

Close the first diamond: **converge** the raw immersion material into a sharp, shared understanding and one well-framed challenge. This is where a pile of notes becomes a problem worth solving.

## Share, then synthesize
- **Sharing discoveries** — each researcher tells the story of what they saw, felt, and learned. Build a common base before analyzing.
- Cluster raw observations into themes; pull out the **nuggets** — the striking insights and surprising quotes worth keeping.

## Model the user
- **Personas** — a few archetypal users built from the research: goals, context, frustrations, behaviors. Grounded in real data, not invented.
- **Empathy map** — for a persona, capture what they *say / think / do / feel* (and pains / gains).
- **Customer journey / experience map** — map the stages, touchpoints, emotions, and pain points across the experience. Spot the moments that matter.

## Prioritize and frame
- **Objectives grid** — line up user needs against project objectives.
- **2×2 matrix** — position findings or opportunities on two axes to reveal where to focus.
- **Priority barometer** — rank the problems/opportunities by importance and effort.
- **Reframing the challenge** — restate the problem as a sharp, actionable challenge. Frame it as an open **"How Might We…"** question — broad enough to invite ideas, narrow enough to be tractable.

## Principles
- Converge to *one* primary challenge; ideation without a frame produces scattered ideas.
- Keep everything traceable to real users — no insight without evidence.

## Exit checklist
Discoveries shared · insights/nuggets clustered · persona(s) built · empathy map + journey mapped · priorities set · challenge reframed as a "How Might We" question. Proceed to `ideation`.

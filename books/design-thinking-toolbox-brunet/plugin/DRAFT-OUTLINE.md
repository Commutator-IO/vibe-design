# Design Thinking Toolbox — skill outline (draft)

Build each skill as `skills/<name>/SKILL.md` following the pattern in `templates/book-template` and the finished `build-sprint` plugin. Each phase skill should offer the concrete *tools* from the book, not just describe the phase.

## `design-thinking` (orchestrator)
Route between the five phases; keep everything anchored to a real user and a real problem.
- **Principles:** user-centered, bias toward action, embrace ambiguity, iterate, "fail fast to learn fast." Phases are non-linear — loop back when needed.
- **Phases:** empathize → define → ideate → prototype → test.

## `empathize`
Understand the user before defining anything.
- **Tools:** qualitative user interviews (open, non-leading), empathy map (says / thinks / does / feels), persona, customer journey map (stages, touchpoints, pain points, emotions).
- **Output:** a grounded picture of who the user is and where they struggle.

## `define`
Turn observations into a sharp problem.
- **Tools:** synthesize interview notes into insights; write a Point-of-View statement ("[user] needs [need] because [insight]"); reframe as "How Might We…" questions.
- **Output:** one well-framed problem statement worth solving.

## `ideate`
Generate and select solutions.
- **Tools:** structured divergence (brainstorming rules, brainwriting, Crazy 8s, worst-possible-idea); then convergence (dot voting, clustering, selection criteria).
- **Guard against:** premature convergence and loudest-voice bias — generate alone, then share.
- **Output:** a shortlist of ideas to prototype.

## `prototype`
Make ideas testable as cheaply as possible.
- **Tools:** choose the lowest-fidelity artifact that answers the open question (paper, wireframe, storyboard, role-play, clickable mock). Build only what the test needs.
- **Output:** a prototype tied to a specific learning goal.

## `test`
Learn from real users.
- **Tools:** user testing sessions (let them use it, think aloud, don't lead); capture feedback neutrally; map results to the POV/HMW.
- **Decision:** iterate (loop back to any earlier phase) or advance.

## Playbook
`playbook/PLAYBOOK.md` should narrate one cycle with an example tool per phase, bilingual FR/EN notes since the source is French.

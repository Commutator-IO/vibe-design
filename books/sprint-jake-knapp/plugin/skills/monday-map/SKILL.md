# Monday — Map & Target

Build a shared picture of the problem and force one decision: exactly what will be built this week. Do NOT open an editor today. Today is thinking. Timebox each step.

## Morning — set direction

1. **Set a long-term goal (optimistic).** Ask why this project exists and where they want to be in 6–12 months. Capture one sentence and keep it visible all week.
2. **List sprint questions (pessimistic).** Ask how it could fail. Turn each fear into a question answerable this week (e.g., "Will users trust it with their data?"). These become Friday's test criteria — save them verbatim.
3. **Make a map.** Draw a simple flowchart of how a user gets from first contact to the goal: actors on the left, the finished goal on the right, 5–15 steps between. Keep it crude. This is the skeleton of what gets built.

_Prompt to offer:_ "Act as a product strategist. Given [idea] and goal [goal], draw a 5–15 step user journey map and list the top 5 failure modes as testable questions."

## Afternoon — narrow to one target

4. **Ask the experts.** Interview anyone who understands the problem (teammates, a domain expert, a real user, or Claude standing in for research), reporter-style, 15–30 min each. Update the goal, questions, and map as you learn.
5. **How Might We notes.** Reframe each problem heard as an opportunity starting with "How might we…", one idea per note. Cluster them and vote for what matters most.
6. **Pick a target.** The Decider circles ONE target customer and ONE moment on the map — the riskiest, most valuable slice. Everything the rest of the week aims here. This single decision is the point of Monday. Push back if the target is broad or safe.

## Exit checklist
Long-term goal posted · sprint questions listed · journey map drawn · experts interviewed · How Might We notes voted · one narrow target chosen by the Decider · Friday recruiting underway. Then move to `tuesday-sketch`.

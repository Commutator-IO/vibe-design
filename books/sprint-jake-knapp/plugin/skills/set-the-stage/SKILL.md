# Set the Stage

Prepare the sprint before Monday. This setup is the biggest predictor of whether the week works. Walk the user through each item; capture the answers in one place they'll reuse all week.

## 1. Pick the right challenge
Use a sprint when the stakes are high, time is short, or the team is stuck. Confirm the challenge is worth five days — a new product, an unproven feature, a workflow to automate, or a prototype to put in front of stakeholders. If it's trivial or already validated, say so.

## 2. Name a Decider
Identify who can make the binding call on target and approach. Decisions won't stick without one. If the Decider can't be present all week, have them appoint a delegate who can and commit to reviewing the prototype before Friday.

## 3. Assign roles
Even a solo builder holds every role. Assign explicitly:
- **Decider** — binding calls on target and approach.
- **Facilitator** — runs the clock, keeps each day timeboxed, blocks scope creep.
- **Maker / Stitcher** — builds with Claude and keeps the pieces coherent.
- **Interviewer** — runs Friday's five interviews; writes the script Thursday.
- **Claude** — expert, researcher, sketch generator, builder. Give it one clear job at a time.

## 4. Block time and space
Reserve five days (~10–5 Mon–Thu, 9–5 Fri). No distractions during focus blocks. Set up one capture surface (whiteboard or a single shared doc) for the goal, questions, map, and decisions.

## 5. Choose rough, fast, flexible tools
Do NOT default to the production stack — it's optimized for quality and fights speed. Recommend a lightweight scaffold, a component library, sample/test data, and a one-command deploy target. Set up a fresh repo now so Thursday isn't spent on plumbing.

## 6. Start recruiting five users
Recruiting five good users is the sleeper task of the week. Start on day one: write a short screener that identifies target users without revealing what you're testing, offer a small incentive, and plan to confirm by email and phone. It takes an hour or two a day.

## Exit checklist
Confirm before Monday: challenge chosen · Decider named · roles assigned · five days blocked · capture surface ready · rough/fast tools + fresh repo set up · user recruiting started. Then move to `monday-map`.
